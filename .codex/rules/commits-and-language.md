# Commits e Linguagem

## Objetivo

Padronizar mensagens de commit e linguagem de documentacao do projeto.

## Regras

- Mensagens de commit em portugues, objetivas e no imperativo.
- Prefixar commits com tipo: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`.
- Descrever impacto funcional no corpo quando necessario.
- Manter textos de interface e docs internas em pt-BR (padrao atual do projeto).

## Anti-padroes

- Commits genericos como `update` ou `ajustes` sem contexto.
- Misturar multiplas mudancas sem relacao em um unico commit.
- Alternar idioma sem necessidade em mensagens e docs do projeto.

## Checklist

- O commit tem tipo valido?
- A mensagem explica a mudanca de forma objetiva?
- O idioma esta consistente com pt-BR?
- Mudancas grandes foram divididas em commits coerentes?
